# 基于1月6日袁老师信息的思考

## EMIS、EDSS 系统分析

应急管理BG现有系统的本质：
- 系统组成：MIS(管理信息系统) + GIS(地理信息系统) + 应急业务 + 工作流引擎等组件
- 功能核心：静态信息的管理 + 静态地理信息的查询与展示 + 针对突发事件的业务流程


## EMIS、EDSS 系统现状

- EMIS 系统现在已经封板
- EDSS 系统按照不同方向（林火、地震、洪涝）构建专题应用
- 目前在做面向基层的应急管理工具


## 辰安的优势与目前的问题

### 辰安的优势

在最开始的时候，通过理念和思想的创新，结合 MIS 和 GIS 等偏通用的技术，开辟了新的赛道。

### 目前的问题

但是，由于路径依赖等问题，公司的信息技术等内容长期没有收到重视，技术积累相对薄弱，到目前核心竞争力不足，陷入了和其他公司打价格战的情况。

### 未来的关键

- 理念创新
- 技术创新
- 产品创新
- 支撑市场

## 新的应急系统需求

应急管理办公室时代，需求相对比较简单、明确：
1. 平时：信息接报、值班排版、上传下达、数据管理（预案、案例等等）
2. 战时：救援方案、决策支持、专家建议


随着2023年中国消防救援局的成立，未来中国的应急管理相关工作，可能会的趋势是：
1. 应急管理部门，主要负责自然灾害监测（不过这个和自然资源部该如何区分？）、安全生产监管、应急能力建设等非战斗工作；
2. 救援力量管理部门
	1. 消防救援局，主要负责救援能力培训建设、非专业类救援（城市等）救援战斗相关工作；
	2. 安全生产救援中心，主要负责生产安全相关的事故救援（下辖93家央国企专业救援队？）（之前做的平台，实现直接到救援队的指挥功能）
	3. 安能集团救援力量，主要负责自然灾害相关的专业救援工作（例如堰塞湖疏通等）
3. 实战救援部门
	1. 各救援队
	2. 实训基地
	3. 培训学校
	4. 相关高校
	5. ……

对于相关系统的需求：
1. 科信司，仍然对于引领应急管理信息化工作，具有主导作用；
2. 各个业务部门，会按照各自的业务需求，以实用为主，开发各单位自己的业务系统；
3. 应急部门与本级消防救援队伍要共建共用应急联合指挥平台，这个该如何把握？

# 数字孪生平台/数字孪生战场的三项创新

## 理念创新

在理念上，数字孪生平台和之前的应急平台有什么区别？

|          数字孪生平台/战场           |         以往应急平台         |
|:------------------------------------:|:----------------------------:|
|           面向实体时空演化           |       面向静态数据管理       |
|           基于Agent的设计            |        面向对象的设计        |
|           面向真实接入数据           |      面向输入/导入数据       |
|            基于模型的推演            |        模型结果的展示        |
|             面向决策生成             |       决策相关数据展示       |
|            面向未来的交互            |        面向桌面/大屏         |
|          面向未来的数据来源          | 面向已有的数据（打补丁录入） |
| 面向新的业务方向（推演、灾害预警……） |      面向以往理解的方向      | 


## 技术创新

实现上述的理念创新，需要研究相关的关键技术。

1. Agent Based Modelling / [Agent Oriented Programming](https://www.sciencedirect.com/science/article/abs/pii/0004370293900349#:~:text=A%20new%20computational%20framework%20is%20presented%2C%20called%20agent-oriented,of%20an%20agent%20is%20called%20its%20mental%20state.)
2. 数据接入（相对成熟，目前研究院已有）
	1. 大数据平台
	2. 时序数据平台
3. 情景推演：实现基于模型的推演而非模型的计算
	1. 基于动力学模型/物理模型等传统方式；
	2. 基于机器学习的模型，例如[基于机器学习的流体力学](https://www.youtube.com/playlist?list=PLMrJAkhIeNNQWO3ESiccZmPssvUDFHL4M)等等；
4. 决策生成：基于深度强化学习的决策生成研究；
5. 三维可视化研究
	1. Unity3D/Unreal 等引擎与 GIS 平台的结合
	2. AR/VR/MR 应用尝试
	3. 人机交互机理的了解与应用
6. 通信相关的内容？融合通信？通信链路？
7. 面向新时代/中国现代化的应急救援理论研究与引入
	1. 指挥体系架构/体制/机制等内容
	2. 相关管理学/数学内容；
	3. 指挥理论相关内容
8. ……



## 产品创新

是否可以是一个云平台，或者偏SAAS的服务？类似于企业微信/钉钉/飞书？
面向时间、空间、层级三个维度，构建一个应用场景矩阵；
产品颗粒化，能够对应不同的矩阵中的具体场景；

- 时间：事前、事发、事中、事后
- 空间：后方、中间、前指、队伍、个人
- 层级：部级、省厅、市局、县局
